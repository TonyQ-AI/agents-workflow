# vt — Version Tracker 项目版本追踪

## 用途

帮助用户在多个 AI 工具之间切换开发时，始终知道 **哪个路径是最新版、完成了哪些功能、最后修改时间是什么**。

## 工作原理

核心数据文件是项目根目录下的 `.vt.json`，所有工具共享同一个数据源。
配套脚本 `vt.py` 提供命令行操作，你也可以直接读写 JSON。

## 使用方式

直接在对话里自然地说，比如：

> "记录一下 resume-calendar 最新版在 D:\\project\\v3，功能有对接高德天气和优化日历渲染"
> "看看我所有项目的最新版本"
> "resume-calendar 的版本历史是怎样的"
> "清理一下 MediaManager 只保留最近3个版本"

AI 会自动调用 vt.py 来执行，或者直接操作 .vt.json 文件。

## vt.py 命令参考

```bash
# 记录版本（新项目自动创建）
python vt.py record <项目名> <路径> [功能描述...]

# 列出所有项目的最新版本
python vt.py list

# 查看某个项目的完整版本历史
python vt.py show <项目名>

# 清理旧版本，只保留最近 N 个
python vt.py prune <项目名> <保留数量>

# 只输出最新路径（脚本集成用）
python vt.py path <项目名>
```

## 直接操作 .vt.json

如果你在 Autoclaw 或其它工具中，也可以直接读写 `.vt.json` 文件。
数据结构：

```json
{
  "projects": {
    "项目名": {
      "versions": [
        {
          "path": "版本路径",
          "updated_at": "2025-06-30 12:00:00",
          "features": ["功能1", "功能2"]
        }
      ]
    }
  }
}
```

版本按记录顺序排列，最后一个是**最新版**。

## 关键规则

1. **相同路径**重复记录 → 只更新 `updated_at` 时间戳，合并新功能
2. **不同路径** → 作为新版本追加，旧版本保留可追溯
3. 功能描述用短语概括即可，不需要长篇大论
4. 每次记录自动记录当前时间，不依赖人工输入
