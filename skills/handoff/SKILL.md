# Reasonix 会话交接

## 概述

将当前会话的关键上下文压缩成一份可读的交接文档，方便另一个 AI 或人类开发者无缝接手。

与 `task_plan.md` / `findings.md` / `progress.md` 三文件配合使用——它们记录"做到哪了"，本技能记录"这里有什么上下文、做了什么决定、有什么坑"。

## 何时使用

- 当前对话太长想开新会话继续
- 要把任务分给另一个 AI 或同事做
- 今天做不完明天继续
- 用户说"交接一下"、"整理一下上下文"、"把进度留给下一个 AI"
- executing-plans 检测到 `/clear` 后自动调用

## 工作流位置

```
对话太长 → 我说"交接一下"
             ↓
        reasonix-handoff
             ↓
        输出交接文档
             ↓
        新会话加载交接文档 + task_plan.md → 无缝继续
```

## 交接文档输出

自动在 `docs/superpowers/handoffs/` 目录下生成 `YYYY-MM-DD-HHmm-项目名-handoff.md`。

### 模板

```markdown
# 会话交接文档

**生成时间：**
**来源会话：** Reasonix - [项目名]
**交接原因：** 对话过长 / 任务交接 / 人工接管

---

## 1. 当前状态

- **项目：** [项目名]
- **阶段：** [规划 / 架构 / 编码 / 测试 / 审查 / 部署]
- **进度：** 已完成 X/Y 个任务，断点在 任务 N/步骤 M
- **阻塞项：** [有/无，具体是什么]

## 2. 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| [技术选型] | [选了A没选B] | [理由] |
| [架构方案] | [方案2] | [理由] |

## 3. 待办事项

### 立即要做
- [ ] [任务描述]
- [ ] [任务描述]

### 需要注意
- [ ] [潜在风险]
- [ ] [待确认事项]

## 4. 发现和坑

| 发现 | 影响 |
|------|------|
| [这个库有兼容问题] | [改用别的] |
| [这部分的测试特别重要] | [需要重点验证] |

## 5. 上下文摘要

[2-3 段话总结当前会话中发生的关键事情]

---

交接文档同时会追加一条引用到 `findings.md`。
```

## 使用方式

### 对话中直接说

```
你：交接一下，我明天继续
→ 生成交接文档 + 更新 progress.md 断点

你：把现在的进度转给小明
→ 生成交接文档，显示内容，可复制给别人
```

### 自动触发

在 executing-plans 中，当检测到需要中断时自动生成交接文档。

### 新会话接手

新会话中加载交接文档 + 检查 task_plan.md：

```
检测到交接文档 docs/superpowers/handoffs/2026-07-03-handoff.md
以及 task_plan.md（已完成 3/8 任务，断点：任务 4/步骤 2）

是否载入上下文继续？[是/否]
```

## 配合已有技能

| 技能 | 合作方式 |
|------|---------|
| **executing-plans** | 中断时自动触发 handoff，恢复时自动加载 handoff |
| **writing-plans** | handoff 引用 task_plan.md 中的任务清单 |
| **findings.md** | handoff 汇总 findings 中的关键发现 |
| **progress.md** | handoff 读取当前进度断点 |
