# 触发词命令集

用户设置了四个关键词触发行为，**任何一次对话都必须遵守**。

## 命令列表

### 1. "醒醒"
立即执行：扫描 `F:\projects\` 下所有 Git 项目目录。
对每个项目显示：名称、最新提交、是否有未提交改动。
然后问用户要打开哪个项目。

### 2. "开新坑" + 项目名
在 `F:\projects\` 下创建文件夹并 `git init`。
完成后说"开好了，进去看看？"并显示项目概况。

### 3. "存档"
在当前项目执行 `git add -A && git commit`。
commit message = "[Reasonix] 从本次对话中提炼的改动总结"。
如果工作区干净就说"没有需要存档的改动"。

### 4. "读档"
在当前项目执行 `git log --oneline -10`，总结最近的提交历史。

## 执行规则

- 任何时候用户说出上述关键词之一，立即执行对应操作
- "醒醒"是最高优先级，**必须立即响应并扫描 projects**
- 如果用户说"醒醒"但上下文表明是别的事，仍然先执行扫描再问需求
